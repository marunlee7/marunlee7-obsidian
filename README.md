# 📚 T-System Vault 구조 안내 (README)

> [!summary] 💡 Quick Summary
> 이 Vault는 **T-시스템(T-System)** 구조를 기반으로 하며, 모든 노트는 T01부터 T05까지의 명확한 레벨과 Status를 통해 관리됩니다.

## 📁 주요 폴더/레벨 역할

* **T01 Series Master:** 프로젝트의 씨앗이 되는 핵심 주제 및 시리즈의 마스터 파일.
* **T02 Article Atom:** 정리된 지식의 기본 단위 (원자). T01 시리즈에 연결되는 구조화된 요약 노트.
* **T03 Dictionary/T04:** 정의, 단어장, 임시 메모 등 T02로 정리되기 전의 파편적인 정보.
* **05_MOCs (T05 System):** Vault의 **심장부**. MOCs(Map of Contents), 대시보드, 각종 인덱스 파일 등 시스템 관리 도구가 위치합니다.

## 📊 Status 관리 시스템 (4-Step)

모든 프로젝트 노트는 다음 4단계의 Status를 가집니다.

| Status              | 의미    | T-시스템 흐름              |
| :------------------ | :---- | :-------------------- |
| **planned**         | 계획/예정 | 자료 수집 후 대기 상태.        |
| **in-progress**     | 진행 중  | 활발하게 요약 및 정리 작업 중.    |
| **waiting/blocked** | 보류/대기 | 외부 요인 등으로 작업이 일시 중단됨. |
| **completed**       | 완료    | 최종 검토 및 정리가 완료됨.      |
