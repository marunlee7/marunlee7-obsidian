# 일반-칼럼 시리즈 내용이 한자 어휘를 다루는 경우



## 🏷️ 예시 5: 칼럼 시리즈 내에서 한자 어휘를 다룰 때 (교차 지식 관리)

이 경우, **1단 개념**은 한자/어휘의 내용이 되고, **2단 영역**은 해당 내용을 포함하는 칼럼 시리즈의 맥락이 됩니다.

|**태그 단계**|**구성 방법 (원칙)**|**태그 예시**|**설명**|
|---|---|---|---|
|**1단: 개념 (Concept)**|칼럼에서 설명하는 **특정 어휘** 또는 한자의 **수준**|`#어휘/법률용어`, `#한자/고급`, `#주제/문화사`|지식의 _주제_ (내용)|
|**2단: 영역 (Domain)**|해당 내용이 연재되는 **칼럼 시리즈의 이름** 또는 **학습 목표**|`#시리즈/시사용어`, `#스터디/어휘력강화`, `#프로젝트/필사`|지식이 _사용되는_ 맥락|
|**3단: 출처 (Source)**|**칼럼이 연재된 언론사** 또는 **특정 기자**|`#언론사/한국경제`, `#저자/홍길동기자`, `#유형/칼럼`|지식의 _근원_|

**👉 YAML 적용 예시 ('한국 경제'에 연재된 '시사 용어' 칼럼 노트):**

이 노트를 통해 특정 시사 용어(예: `공법(公法)`)를 학습하고, 이 용어가 어떤 칼럼 시리즈에 속하며, 누가 썼는지까지 구조적으로 저장할 수 있습니다.

YAML

```
---
# 1단: 개념/주제
concept: [어휘/법률용어, 한자/공법(公法)]

# 2단: 영역 (칼럼 연재 맥락)
domain: [시리즈/시사용어, 스터디/어휘력강화]

# 3단: 출처
source: [언론사/한국경제, 저자/홍길동기자]
---
```

### 💡 이 구조의 장점

1. **개념 중심 검색:** `#어휘/법률용어`만 검색하면, 이 칼럼뿐 아니라 다른 교재나 문서에서 얻은 법률 용어 정보까지 **모두** 모아서 볼 수 있습니다.
    
2. **영역 중심 검색:** `#시리즈/시사용어`만 검색하면, 이 칼럼 시리즈에서 다룬 모든 주제(경제, 정치, 한자 등)의 노트를 **순서대로** 모아서 볼 수 있습니다.
    

이처럼 지식이 교차될 때, 각 지식의 역할을 명확히 구분해 주는 것이 바로 3단 태그 시스템의 핵심적인 역할입니다.
