# 네스티드 태그


## 📘 요약

네스티드 태그(Nested Tags)는 메모 앱 **옵시디언(Obsidian)** 에서 노트들을 계층적이고 체계적으로 분류하고 검색하는 데 매우 유용한 기능입니다.

옵시디언에서 네스티드 태그는 **슬래시($/$)** 를 사용하여 정의하며, 관련된 태그들을 **트리 구조**로 조직할 수 있게 해줍니다.

---

## 1. 옵시디언 네스티드 태그의 정의 및 작성법

==옵시디언에서 태그 이름에 슬래시($/$)를 사용하면 중첩된 태그(계층적 태그)가 됩니다.==

- **작성법:**

```
#상위태그/하위태그/하위의하위태그
```

- **예시:**
    
- `#업무/프로젝트A/아이디어`
	
- `#독서/요약/아티클`
	
- `#상태/todo`
	
- `#상태/진행중`
	
- `#상태/완료`
        

## 2. 네스티드 태그의 주요 장점

네스티드 태그는 단순한 태그 나열(Flat Tags)을 넘어 노트 관리에 여러 이점을 제공합니다.

|**장점**|**설명**|**예시**|
|---|---|---|
|**계층적 구조**|태그를 논리적인 그룹으로 분류하여 **태그 패널(Tags pane)**을 깔끔하게 유지합니다.|`#업무/프로젝트A` 와 `#업무/프로젝트B`를 모두 `#업무`라는 큰 범주 아래에서 볼 수 있습니다.|
|**쉬운 검색 및 필터링**|상위 태그를 검색하면 모든 하위 태그가 자동으로 포함되어 검색됩니다.|**`tag:#업무`**를 검색하면 `#업무/프로젝트A`, `#업무/프로젝트B` 등 `#업무`로 시작하는 모든 태그가 달린 노트를 한 번에 찾을 수 있습니다.|
|**개념의 구체화**|주제나 상태를 더 명확하고 구체적으로 정의하는 데 도움을 줍니다.|단순한 `#완료` 대신, `#상태/완료`를 사용하여 상태 관련 태그임을 명확히 할 수 있습니다.|

## 3. 태그 사용 팁: 폴더, 링크와의 차이점

옵시디언에서는 **폴더, 내부 링크, 태그** 세 가지 방식으로 노트를 정리할 수 있으며, 네스티드 태그는 그중 태그의 유연성을 극대화합니다.

|**구분**|**목적 및 특징**|**네스티드 태그 사용 시점**|
|---|---|---|
|**폴더**|**파일 시스템**의 계층 구조를 따르며, 파일을 물리적으로 보관하는 용도입니다.|**노트의 유형** (예: `Literature Notes`, `Daily`) 등, **변하지 않는 분류 기준**에 주로 사용.|
|**내부 링크**|**노트 간의 연결 관계**를 만듭니다. 지식 간의 유기적인 연결(MOC)에 필수적입니다.|**노트 자체가 개념**이 될 때 (예: [[아리스토텔레스]], [[양자역학]]), 또는 **세부적인 맥락**이 중요할 때 사용.|
|**태그**|**노트의 속성, 상태, 주제** 등을 분류하는 데 사용됩니다. 하나의 노트에 여러 태그를 쉽게 붙일 수 있습니다.|**상태**(`#todo`, `#진행중`), **유형**(`#아티클`), **광범위한 주제**(`#생산성/옵시디언`) 등 **다차원적인 분류**가 필요할 때 사용.|

네스티드 태그는 폴더의 계층성과 태그의 유연성을 결합하여, 폴더 구조에 구애받지 않는 **가상적인 계층 구조**를 만들어 노트를 관리할 수 있게 해줍니다.
