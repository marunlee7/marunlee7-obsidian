# 파라 한자 학습 프론트매터


**학습 상태, 난이도, 복습 주기**에 초점을 맞춘 태그 분류 및 프론트매터(필드) 재배치는 옵시디언 DataView를 활용하여 학습 큐(Queue)를 관리하는 데 가장 핵심적인 요소입니다.

이전 논의 내용을 통합하고, 관리 효율을 극대화할 수 있도록
항목을 **세 가지 주요 영역**으로 분류하여 제안해 드립니다.

## 🚀 학습 관리 최적화 프론트매터 필드 제안

| **영역**                                          | **필드명 (Key)**       | **추천 값 형식**            | **초점 및 용도**                                                                                             |
| ----------------------------------------------- | ------------------- | ---------------------- | ------------------------------------------------------------------------------------------------------- |
| **I. 복습 주기 (Spaced Repetition)**                | `last_review`       | `YYYY-MM-DD` (날짜)      | **마지막 복습 시점**. 다음 복습일(`next_review`) 계산의 기준이 됩니다.                                                       |
|                                                 | `next_review`       | `YYYY-MM-DD` (날짜)      | **다음 복습 예정일**. DataView로 이 날짜가 지난 노트만 필터링하여 복습 목록(큐)을 만듭니다.                                             |
|                                                 | `review_count`      | 숫자 (정수)                | **총 복습 횟수**. 노출 빈도를 추적합니다.                                                                              |
쉬움(y). 현재 복습 결과에 따른 _상태_를 나타냅니다.           |
|                                                 | `is_mastered`       | `true` / `false` (불리언) | **최종 마스터 여부**. (기존 `checkbox`나 `read`를 대체) `level`이 4이고 `review_count`가 일정 횟수 이상일 때 `true`로 설정하여 관리합니다. |
| **III. 난이도 및 분류 (Difficulty & Categorization)** | `tags`              | `한자어휘/한자암기박사`          | **출처, 카테고리 분류**. (필요시 태그를 세분화하여 난이도를 나타낼 수 있으나, `level`과 `difficulty_rating`을 추천)                       |
|                                                 | `difficulty_rating` | 숫자 (1~10)              | **절대적인 난이도**. `level`이 복습 _결과_라면, 이것은 단어 자체의 _본질적 난이도_를 나타냅니다. (예: 10점은 매우 희귀하거나 어려움)                   |

---

### 💡 태그 및 필드 활용 전략

#### 1. 난이도 관리 (Level vs. Rating)

|**필드**|**역할**|**활용 예시**|
|---|---|---|
|**`level`**|**동적 상태**|DataView로 `WHERE level <= 2` (어려운 노트)를 추출하여 짧은 주기로 복습할 노트 목록을 만듭니다.|
|**`difficulty_rating`**|**정적 난이도**|`difficulty_rating: 9`인 노트는 `next_review`가 한참 남았더라도 가끔 특별 복습 대상으로 삼을 때 사용합니다.|

#### 2. 복습 주기 관리 (The Review Queue)

DataView를 사용할 때, 복습이 필요한 노트 큐(Queue)를 만드는 핵심 쿼리는 다음과 같습니다.

today(e)
> 
> 오늘 날짜보다 복습 예정일이 과거인 모든 노트를 보여줍니다.

#### 3. 태그(`tags`) 활용

태그는 복잡한 복습 상태보다는 **카테고리 분류**와 **출처 관리**에만 집중하는 것이 좋습니다.

- `tags:`
    
    - `- 한자어휘/한자암기박사` (출처/분류)
        
    - `- 카테고리/형법` (주제 분류)
        
    - `- 상태/보류` (특수한 상태 분류)
        

### 📝 최종 프론트매터 템플릿

이전 '필법' 예시에 위의 제안을 모두 적용한 최종 템플릿입니다.

YAML

```
---
# 1. 기본 정보
title: 필법(筆法)
aliases:
  - 필법(筆法)
tags:
  - 한자어휘/한자암기박사 
created: 2025-11-10 

# 2. 복습 주기 (Spaced Repetition)
last_review: 2025-11-10
next_review: 2025-11-11 
review_count: 1

# 3. 학습 상태 및 난이도
Easy(4)
level: 1 
어려움(0)
difficulty_rating: 4 
is_mastered: false # 최종 학습 완료 여부

# 4. 단어 및 한자 정보
korean_sound: 필법
korean_meaning: 글씨나 그림을 그리는 기술적인 방법이나 규칙.
korean_mnemonic: 붓(筆)을 다루는 법(法)
ndic: https://hanja.dict.naver.com/#/search?query=筆法
links:
  - "[[서예-운필법]]" 
hanja_list: 
  - 筆
  - 法
---
```

이 템플릿은 학습 상태, 난이도, 복습 주기 세 가지 초점에 완벽히 맞춰져 있으며, DataView를 통한 자동 복습 큐 관리에 가장 효율적입니다.

혹시 이 템플릿을 활용하여 **오늘 복습할 노트 목록을 보여주는 DataView 쿼리 예시**가 필요하신가요?
```
---
aliases: 필법(筆法)
created: 2025-11-10
modified: 2025-11-10
tags:
  - area/learning
  - type/concept
  - source/hanja_book
  - difficulty/medium
mastery_level: 3
next_review: 2025-11-17
ndic: https://hanja.dict.naver.com/#/search?query=筆法
sound: 필법
meaning: 글씨나 그림을 그리는 기술적인 방법이나 규칙.
related_chars:
  - [[筆]]
  - [[法]]
context: "이 서예가는 붓의 움직임에 깊은 필법(筆法)을 담아낸다."
---
```

이러한 필드와 태그를 추가하면 Dataview 플러그인을 활용하여 **"복습일이 지난 난이도 Medium 이상의 한자"** 목록을 자동으로 생성하여 학습 효율을 극대화할 수 있습니다.
