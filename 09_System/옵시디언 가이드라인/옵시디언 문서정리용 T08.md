# ⚙️ 옵시디언 문서정리용 T08 (T08)

네, 사용자님의 제안은 **T-시스템의 논리적 확장** 측면에서 매우 적절하며 체계적입니다.
[[T08 워크플로우 앱 연구 레벨 확정]] 과 같은 내용


## ✅ T08: 워크플로우/앱 연구 레벨 확정

| **T-레벨 범위** | **역할**              | **예시**                       |
| ----------- | ------------------- | ---------------------------- |
| **T01-T05** | **핵심 콘텐츠 / 지식 관리**  | Series, Atoms, MOCs 등        |
| **T06-T08** | **워크플로우 / 프로젝트 관리** | 프로젝트, 목표, **앱 공부/설정**        |
| **T09**     | **Vault 시스템 / 설정**  | `09_System_Index`, 플러그인 설정 등 |

**T08**을 **'워크플로우 / 도구 학습 (Tooling/Workflow Study)'** 레벨로 지정하는 것은 논리적입니다. 이 노트들은 지식 자체는 아니지만, 지식을 생산하는 **환경**에 대한 중요한 문서이기 때문입니다.

---

# 🛠️ T08 템플릿 표준 제안 (App Study)

T08은 시스템 파일의 성격이 강하지만, 정보의 출처(Source)가 필요하므로, 기존 T-시스템의 **[정의] → [구조] → [상태] → [출처]** 표준 순서를 유지합니다.

## 1. T08 YAML 메타데이터 표준 정의

|**필드명**|**T08의 역할**|**T08 기본값**|
|---|---|---|
|`level`|T08_Workflow|T08_Workflow|
|`file_role`|`tool_config` 또는 `app_study`|`tool_config`|
|`source_name`|앱/툴 이름 (예: Obsidian Quickadd)|(Quickadd 입력)|
|`source_type`|앱 문서, 유튜브 튜토리얼 등|`web` 또는 `video`|
|`parent`|해당 앱의 메인 MOC (예: `[[Obsidian Hub]]`)|(Quickadd 입력)|
|`status`|설정 진행 상태|`"draft"`|

## 2. T08 Quickadd 템플릿 제안

이 구조를 바탕으로, T08 파일을 생성할 수 있는 Quickadd 템플릿을 제안합니다.

Markdown

```
---
title: {{value:T08 앱/툴 설정 주제}}

file_role: "tool_config"

parent: "{{value:상위 앱/도구 MOC (예: [[Obsidian Hub]])}}"
word_type: ""

process: "raw"

source_name: "{{value:주요 출처 이름 (강의/문서 이름)}}"
source_type: "{{value:app_doc,video,web}}"  <-- T08에 특화된 출처 목록
source_author: "{{value:출처 저자/제작자 (예: Tutorial Channel)}}"

references:
  - "{{value:주요 출처 URL (필요시 입력)}}"
tags:
  - T08/tool
  - app/obsidian
---
# ⚙️ {{title}} (T08)
```

이 T08 템플릿을 사용하시면, 워크플로우에 대한 정리도 T-시스템의 확립된 규칙을 따르게 됩니다. 이 템플릿으로 T-시스템 확장을 진행하시겠습니까?
